## 1. Общая Docker Overlay-сеть (выполнить на одном из серверов-менеджеров)

```bash
docker swarm init --advertise-addr 10.0.0.11
# На остальных серверах:
docker swarm join --token <TOKEN> 10.0.0.11:2377

docker network create --driver overlay --attachable kafka-net
```

## 2. ZooKeeper Ensemble (3 ноды)

### Server-1: `/opt/kafka-cluster/docker-compose.zk.yml`

```yaml
version: '3.8'

services:
  zookeeper-1:
    image: confluentinc/cp-zookeeper:7.6.0
    hostname: zookeeper-1
    container_name: zookeeper-1
    ports:
      - "2181:2181"
      - "2888:2888"
      - "3888:3888"
    environment:
      ZOOKEEPER_SERVER_ID: 1
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000
      ZOOKEEPER_INIT_LIMIT: 5
      ZOOKEEPER_SYNC_LIMIT: 2
      ZOOKEEPER_SERVERS: zookeeper-1:2888:3888;zookeeper-2:2888:3888;zookeeper-3:2888:3888
    volumes:
      - zk-data:/var/lib/zookeeper/data
      - zk-logs:/var/lib/zookeeper/log
    networks:
      - kafka-net
    restart: unless-stopped

volumes:
  zk-data:
  zk-logs:

networks:
  kafka-net:
    external: true
```

## 3. Kafka Brokers (3 ноды)

### Server-1: `/opt/kafka-cluster/docker-compose.kafka.yml`

```yaml
version: '3.8'

services:
  kafka-1:
    image: confluentinc/cp-kafka:7.6.0
    hostname: kafka-1
    container_name: kafka-1
    ports:
      - "9092:9092"
      - "29092:29092"
      - "9999:9999"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper-1:2181,zookeeper-2:2181,zookeeper-3:2181
      
      # Слушатели: внутренний (docker) и внешний (для клиентов)
      KAFKA_LISTENERS: INTERNAL://kafka-1:29092,EXTERNAL://0.0.0.0:9092
      KAFKA_ADVERTISED_LISTENERS: INTERNAL://kafka-1:29092,EXTERNAL://10.0.0.11:9092
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: INTERNAL:PLAINTEXT,EXTERNAL:PLAINTEXT
      KAFKA_INTER_BROKER_LISTENER_NAME: INTERNAL
      
      # Отказоустойчивость и репликация
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 3
      KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: 3
      KAFKA_TRANSACTION_STATE_LOG_MIN_ISR: 2
      KAFKA_MIN_INSYNC_REPLICAS: 2
      KAFKA_DEFAULT_REPLICATION_FACTOR: 3
      
      # Логи и retention
      KAFKA_LOG_RETENTION_HOURS: 168
      KAFKA_LOG_SEGMENT_BYTES: 1073741824
      KAFKA_LOG_RETENTION_CHECK_INTERVAL_MS: 300000
      
      # JMX (опционально, для мониторинга)
      KAFKA_JMX_PORT: 9999
      KAFKA_JMX_HOSTNAME: 10.0.0.11
      
      # Количество разделов для __consumer_offsets
      KAFKA_OFFSETS_TOPIC_NUM_PARTITIONS: 50
      
      # Auto create topics (в production лучше false)
      KAFKA_AUTO_CREATE_TOPICS_ENABLE: "false"
      
    volumes:
      - kafka-data:/var/lib/kafka/data
    networks:
      - kafka-net
    restart: unless-stopped
    depends_on:
      - zookeeper-1  # если на одном сервере, иначе убрать

volumes:
  kafka-data:

networks:
  kafka-net:
    external: true
```

## 4. Проверка кластера

```bash
# Проверка ZooKeeper ensemble
docker exec -it zookeeper-1 zookeeper-shell zookeeper-1:2181 ls /brokers/ids

# Проверка Kafka
docker exec -it kafka-1 kafka-broker-api-versions --bootstrap-server kafka-1:29092

# Список топиков
docker exec -it kafka-1 kafka-topics --bootstrap-server kafka-1:29092 --list

# Создание тестового топика (3 партиции, 3 реплики)
docker exec -it kafka-1 kafka-topics \
  --bootstrap-server kafka-1:29092 \
  --create --topic test-topic \
  --partitions 3 --replication-factor 3

# Описание топика
docker exec -it kafka-1 kafka-topics \
  --bootstrap-server kafka-1:29092 \
  --describe --topic test-topic
```

## 5. Альтернатива: docker-compose без Swarm (через прямые IP)

Если Docker Swarm не подходит, используйте **host network mode** на каждом сервере, заменив в каждом `docker-compose.kafka.yml`:

```yaml
    network_mode: host
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: 10.0.0.11:2181,10.0.0.12:2181,10.0.0.13:2181
      KAFKA_LISTENERS: PLAINTEXT://0.0.0.0:9092
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://10.0.0.11:9092
      # ... остальные параметры
```

## 6. Ключевые параметры отказоустойчивости

| Параметр | Значение | Смысл |
|----------|----------|-------|
| `replication-factor=3` | 3 | Каждая партиция хранится на 3 брокерах |
| `min.insync.replicas=2` | 2 | Минимум 2 реплики должны подтвердить запись |
| `offsets.topic.replication.factor=3` | 3 | Репликация служебного топика смещений |
| `transaction.state.log.replication.factor=3` | 3 | Репликация логов транзакций |
| `transaction.state.log.min.isr=2` | 2 | Минимум ISR для транзакций |

> **Важно:** Для полной отказоустойчивости при создании топиков всегда указывайте `--replication-factor 3` и `--partitions >= 3`.

```bash
#!/bin/bash
# deploy.sh — запускать на каждом сервере

mkdir -p /opt/kafka-cluster
cd /opt/kafka-cluster

# 1. Создать сеть (на сервере-1)
# docker network create --driver overlay --attachable kafka-net

# 2. Запуск ZooKeeper
docker compose -f docker-compose.zk.yml up -d

sleep 10

# 4. Запуск Kafka
docker compose -f docker-compose.kafka.yml up -d

echo "Кластер запущен. Проверка:"
docker ps
```
