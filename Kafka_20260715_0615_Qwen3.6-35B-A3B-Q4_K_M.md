### 🔁 Часть 2: JSON → ZooKeeper формат: есть ли возможность?

#### ❌ Краткий ответ
**Нет, и не нужно.** Прямое преобразование JSON в "ZooKeeper формат" не существует и **не рекомендуется**.

1. **Kafka не хранит конфиги в JSON внутри ZooKeeper.**

3. **Современный подход:**
   Kafka предоставляет официальные CLI для экспорта/импорта конфигов. JSON используется только как промежуточный формат для CI/CD или бэкапов.

### ⚠️ Критичные нюансы для Docker + JMX

| Проблема | Решение |
|----------|---------|
| **Динамические RMI-порты** | JMX по умолчанию выделяет случайный порт для RMI server. В Docker это ломает подключение. Фиксируем через `KAFKA_JMX_OPTS` (`-Dcom.sun.management.jmxremote.port=9999`). |
| **Network Alias vs IP** | JMX Exporter подключается к `kafka_${BROKER_ID}:9999`. Убедитесь, что `SERVER_IP` в `.env` соответствует хосту/виртуалке, а не `localhost` внутри контейнера. |
| **Безопасность** | В примере `authenticate=false` и `ssl=false` для простоты. В продакшене добавьте `-Dcom.sun.management.jmxremote.authenticate=true` и настройте `jmx_exporter` на использование SSL-коннекта. |
| **Кардинальность метрик** | `topic` в конфигах JMX exporter может создавать тысячи метрик. Используйте `drop`/`include` правила, если кластер >50 топиков. |
